### 1.2 Synthese

**12 occurrences** de padding/margin hors tokens identifies dans 6 modules CSS. Patterns recurrents :

1. **Valeur 0.7rem** : utilisee 3+ fois (`.tab`, `.input`, `.dropdown-trigger`) mais absente de l'echelle `--space-*`. Necessiterait soit `--space-2 : 0.75rem` soit alignement sur `--space-sm` (0.5rem).
2. **Valeur 0.9rem** : utilisee dans `.kanban-card` et `.backlog-item` — zone grise entre `--space-sm` et `--space-md`.
3. **Valeur 1.2rem** : utilisee dans `.card-compact` et `.backlog-item` — entre `--space-md` (1rem) et `--space-lg` (1.5rem), aucun token existant.
4. **Tokenisation partielle** : plusieurs valeurs numeriquement correctes (2rem = `--space-xl`) mais hardcodees au lieu d'utiliser `var(--space-xl)`.

L'echelle actuelle (`xs=0.25`, `sm=0.5`, `md=1`, `lg=1.5`, `xl=2`) manque de granularite entre `sm` et `md` (rien a 0.75rem) et entre `md` et `lg` (rien a 1.25rem).

| ID   | Fichier CSS                    | Ligne | Probleme                                                                                | Severite | Sous-issue |
|------|-------------------------------|-------|----------------------------------------------------------------------------------------|----------|------------|

### 2.4 Synthese

Deux classes de problemes distincts :
1. **SVG-in-SVG sprite** (P-01) : overhead structural dans le fichier sprite — chaque `<symbol>` emballe un `<svg>` complet avec `width`/`height`. Le pattern correct est `<symbol>` contenant directement les `<path>`, sans `<svg>` interne. Impact : ~21KB actuel, potentiellement ~15KB apres nettoyage.
2. **Couleurs hardcodees dans SVG/CSS** (P-02, P-09) : le chevron de `select.input` est un `data:uri` avec une couleur hex fixe (`#94a3b8`). Ne s'adapte pas au theme ou au mode dark/light. Finding prioritaire car affecte tous les `<select>` du DS et des projets consommateurs.

### 3.2 Analyse patterns recurrents

**25 occurrences de `transition: all`** dans 12 modules CSS (buttons: 4, forms: 5, templates: 3, navigation: 2, cards: 2, lists: 2, interactive: 2, badges/notifications/feedback/tracker/modals: 1 chacun).

**Pattern critique : cards + backdrop-filter + transform hover** (F-05, F-06) :

```css
/* Pattern actuel (risque flicker) */
.card {
  backdrop-filter: blur(20px);      /* force GPU layer */
  transition: all 0.4s;             /* re-trigger sur TOUTES proprietes */
}
.card:hover {
  transform: translateY(-6px);      /* change compositing layer = repaint */
  box-shadow: var(--shadow-lg);     /* 2eme changement sur meme frame */
}
```

`backdrop-filter` force le navigateur a creer un stacking context GPU. Quand `transition: all` est applique et que `transform` change au hover, le navigateur doit recalculer le compositing du element AND ses enfants. Sur certains GPU/drivers (notamment Safari mobile, Firefox < 103), ca provoque un flash blanc/gris d'une frame (~16ms) visible surtout sur les ecrans a 60Hz.

**Correctif candidat** :
```css
/* Correctif (reference uniquement, Phase 3) */
.card {
  backdrop-filter: blur(20px);
  will-change: transform;           /* pre-alloue le layer GPU */
  transition: transform 0.4s var(--ease-standard),
              box-shadow 0.4s var(--ease-standard),
              border-color 0.4s var(--ease-standard);
}
```

L'inspection DevTools Performance n'a pas pu etre menee directement (environment CI headless). Les findings F-01 a F-12 sont bases sur l'analyse statique du CSS et les patterns connus de compositing layer thrashing.
