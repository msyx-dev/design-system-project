# Audit adversarial — Design System msyx

> **Date** : 2026-06-13 · **Cibles** : DS CSS `@ds-version 2.65.2` + `@msyx-dev/react 3.0.0-alpha.6`
> **Méthode** : 8 auditeurs adversariaux spécialisés (analyse statique, sans rendu navigateur) + contre-vérification adversariale indépendante de chaque constat majeur (agent chargé de *réfuter*).
> **Volumétrie** : 161 constats remontés · 64 constats majeurs soumis à contradiction → **42 confirmés, 21 nuancés, 1 réfuté, 0 invalidé sur le fond**.

## Légende

- **Sévérité** : 🔴 bloquant · 🟠 majeur · 🟡 mineur · ⚪ nit. *Lorsqu'un constat a été contre-vérifié, la sévérité affichée est celle **corrigée** par l'adversaire ; la sévérité d'origine est indiquée si elle diffère.*
- **Nature** : `FAIT` = vérifiable dans les sources (avec citation) · `OPINION` = jugement/recommandation.
- **Vérif.** : verdict de contre-vérification adversariale — `confirmé` / `nuancé` (fait vrai, gravité ou portée ajustée) / `réfuté` (qualification invalidée). Vide = non soumis (mineur/nit/opinion non contre-vérifiés).
- ⚠️ *Aucun accès au rendu navigateur : tout constat visuel est marqué « non vérifié au rendu » et plafonné en confiance.*

## A. Hypothèses retenues

- Pas d'accès au rendu navigateur — audit 100 % statique (lecture des sources).
- `tokens.css` et `themes.css` sont les *sources* de tokens (exclus du scan « valeur hardcodée »), mais leur divergence mutuelle est elle-même auditée.
- La règle projet « DS = source unique, l'app consomme TOUT » est prise au mot → parité CSS↔React = exigence ; les manques React sont néanmoins qualifiés *dette d'alpha* quand justifié.
- Le « registre = 101 composants » est traité comme une affirmation à vérifier (résultat : ~77 composants réels + 24 entrées de modules CSS).

## B. Synthèse exécutive

Le DS est **riche et mature** (≈77 composants, theming 3×2, VR bloquante, lint anti-hardcode, suite React de 136 tests comportementaux), mais il **viole en interne plusieurs règles qu'il impose aux consumers** et sa **chaîne de distribution est cassée à froid** : un dev tiers qui suit la doc obtient un DS amputé (pas de fonts, thèmes périmés, icônes manquantes, barrels en 404). Les **invariants documentaires ont massivement dérivé** (5 décomptes de composants contradictoires, 3 décomptes de modules, versions désync sur 3 niveaux) faute de **garde-fou CI**. Le **package React (≈7 % de parité)** rend « consomme TOUT » inapplicable et invente déjà des classes hors-DS en alpha.

### Axe 1b — Theming & Naming

> _Synthèse auditeur_ : Verdict axe 1b : PROBLÈME.
