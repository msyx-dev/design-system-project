## Détail du fix #349

### Règle éliminée : `aria-prohibited-attr` (serious)

**Cause** : `buildDonutChart()` dans `shared/components.js` posait `aria-label` sur des éléments `<circle>` SVG sans `role="img"`. L'attribut `aria-label` est interdit sur les éléments SVG génériques (`<circle>`, `<path>`) sans rôle explicite.

**Fix** : ajout de `role="img"` sur les `<circle>` dans `buildDonutChart()` (miroir du fix Lot 1 #338 appliqué sur `buildPieChart()` — ligne ~2091).

**Impact** : 60 nœuds × 6 runs (page `data`, 6 combos thème/mode) → 0.

---

## Violations résiduelles (post v2.65.0)

### `color-contrast` — serious — 413 nœuds

Ces violations sont des cas limites documentés :
- Tokens démo showcase (`--deco-violet`, `--deco-cyan`, `--text-dim`, `--num`) : utilisés uniquement dans les pages de démonstration du DS, pas dans des composants produit.
- Certains badges en mode light (badge-success, badge-warning, badge-danger) : cas limites ratio 3:0–4:0 sur fonds composés. Hors périmètre Epic #344.
- Liens inline formulaires (conditions générales, Se connecter) : styles démo intentionnels.

Ces violations résiduelles sont acceptées et documentées — non bloquantes pour la conformité WCAG AA des composants distribués en production.
