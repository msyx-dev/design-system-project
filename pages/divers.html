Accordion

Sections repliables pour FAQ ou documentation.

Quelle stack est utilisee ?
Next.js 14 (App Router) + TypeScript en frontend, FastAPI en backend, PostgreSQL + Prisma pour la base de donnees, Caddy en reverse proxy et Docker pour l'isolation.
Comment deployer un projet ?
Utilisez la commande /deploy dans Claude Code. Le skill deploy-caddy configure automatiquement le sous-domaine, le certificat HTTPS et l'auth gate.
Comment personnaliser la charte ?
Modifiez les variables CSS dans le bloc :root. Les couleurs, gradients, rayons et ombres sont tous parametrables via ces tokens.
L'auth gate est-elle obligatoire ?
Oui, par defaut tous les projets deployes sur *.msyx.fr sont proteges. Pour la desactiver, retirez le bloc basic_auth du Caddyfile et rechargez Caddy.

Rendu Markdown / Prose

Classe .prose : conteneur scopé qui stylise du HTML natif déjà rendu (sortie markdown). CSS-only, zéro JS.

Échantillon complet

Design System msyx

Le design system msyx.fr est la source de vérité pour tous les composants UI. Il couvre typographie, couleurs, espacements, et composants interactifs — tout via des var(--token) CSS.

Principes clés

  • Tokens-first — aucune valeur hexadécimale codée en dur
  • Mobile-first — media queries min-width uniquement
  • A11y WCAG AA — contraste 4.5:1, cibles 44px, aria-* complets

Installation

  1. Importer styles.css dans votre <head>
  2. Ajouter nav.js et components.js en fin de <body>
  3. Définir data-theme et data-mode sur <html>

La règle d'or : si un composant est absent du DS, créez-le ici d'abord — jamais dans le projet consommateur.

Exemple de code

/* Importer le design system */
@import "/shared/styles.css";

/* Utiliser la classe prose */
<div class="prose">
    /* Le HTML rendu depuis markdown ici */
</div>
