Durations

3 durées canoniques pour les transitions et animations courtes. Préférer toujours un token, jamais une valeur hardcodée.

--duration-fast 150ms · cubic-bezier(0.4, 0, 0.2, 1)
--duration-base 200ms · cubic-bezier(0.4, 0, 0.2, 1)
--duration-slow 350ms · cubic-bezier(0.4, 0, 0.2, 1) 
/* Usage */
transition: all var(--duration-base) var(--ease-standard);
animation: fadeIn var(--duration-slow) var(--ease-spring);

Easings

2 courbes cubic-bezier : standard pour les transitions classiques, spring pour les apparitions avec overshoot léger (effet rebond).

--ease-standard cubic-bezier(0.4, 0, 0.2, 1)
--ease-spring cubic-bezier(0.34, 1.56, 0.64, 1) 
/* standard — transitions classiques (entrées, sorties, états) */
transition: transform var(--duration-base) var(--ease-standard);

/* spring — apparitions et feedbacks positifs */
animation: scaleIn var(--duration-slow) var(--ease-spring);

Patterns canoniques

6 patterns prêts à copier. Chaque pattern utilise uniquement des tokens — zéro valeur hardcodée. Copier la classe CSS et le keyframe associé dans votre projet.

1. Fade-in

Fade In
/* CSS */
.fade-in {
    animation: motionFadeIn var(--duration-base) var(--ease-standard);
}
@keyframes motionFadeIn {
    from { opacity: 0; }
    to   { opacity: 1; }
}

2. Slide-up

Slide Up
/* CSS */
.slide-up {
    animation: motionSlideUp var(--duration-base) var(--ease-standard);
}
@keyframes motionSlideUp {
    from { opacity: 0; transform: translateY(12px); }
    to   { opacity: 1; transform: translateY(0); }
}

3. Scale-in

Scale In
/* CSS */
.scale-in {
    animation: motionScaleIn var(--duration-base) var(--ease-standard);
}
@keyframes motionScaleIn {
    from { opacity: 0; transform: scale(0.92); }
    to   { opacity: 1; transform: scale(1); }
}

5. Skeleton shimmer

Réutilise @keyframes shimmer 
/* CSS — réutilise le keyframe shimmer global */
.skeleton-line {
    background: linear-gradient(
        90deg,
        var(--surface-solid) 25%,
        var(--surface-light) 50%,
        var(--surface-solid) 75%
