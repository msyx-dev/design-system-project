Usage — Auth Flows

Password strength : .login-strength[data-strength-target="inputId"] + .login-strength-fill[data-level="0-4"]. Niveaux : 1 Faible, 2 Moyen, 3 Bon, 4 Fort.

Steps : [data-auth-container] contient des .login-step[data-step="1"]. Navigation via [data-auth-step-to="N"].

CGU : .login-cgu — checkbox + texte avec liens.

LoginScreen : .login-card--internal-only / --public-multi-providers / --internal-with-fallback. Bouton primary .login-authentik-btn toujours présent (apps internes msyx).

Slots providers : .login-providers > .login-provider-btn--{google|apple|microsoft|github}. Présentationnel — le câblage OAuth incombe au consumer (la fédération vit dans Authentik, ADR-016).

JS : initAuthFlows().

      Usage

      Usage

      Password avec révélation

      Champ mot de passe avec toggle œil (show/hide). Réduit les erreurs de saisie mobile. Combinable avec .login-strength.

      Vide
      Rempli
      + Force du mot de passe
      Usage

      Wrapper .password-field (position relative) autour d'un .input[type=password] + bouton .password-toggle[data-password-toggle]. JS : initPasswordToggle() bascule type, l'icône et aria-pressed/aria-label.

      Liaison input ↔ bouton : aria-controls pointe l'id de l'input (sinon le bouton cible le .input[type=password] du .password-field parent).

      Combinable avec .login-strength (composant indépendant).
