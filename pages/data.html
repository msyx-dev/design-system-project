Virtual list

Liste fenêtrée (windowing) — seules les lignes visibles sont rendues dans le DOM, quel que soit le nombre total d'éléments. Couche de présentation pure : le consumer fournit les données déjà triées/filtrées.

Démo — 1000 lignes, DOM ~15 nœuds

1000 lignes logiques, DOM ~15 nœuds rendus (fenêtre visible + overscan) ; hauteur de ligne fixe.

Usage

Structure : .virtual-list avec data-vlist-count (nombre total logique) contenant .virtual-list-viewport (hauteur fixe --vlist-height, scrollable). Le JS génère un spacer haut, les lignes rendues (.virtual-list-row, hauteur fixe --vlist-row-h) et un spacer bas, pour préserver la hauteur totale de la scrollbar.

Fenêtrage : au scroll (throttlé via requestAnimationFrame), calcule first = floor(scrollTop / rowH) - overscan et visibleCount = ceil(viewportH / rowH) + 2×overscan (overscan = 5), clamp aux bornes. Ne re-rend que si la fenêtre logique change.

JS : initVirtualList() — démo auto-générée (liste déterministe « Élément #N », pas de random). Point d'extension consumer : window.__vlistRenderRow(index) pour brancher son propre tableau de données + renderer. Zero fetch, zero tri/filtre (fournis par le consumer). aria-rowcount = total logique, aria-rowindex sur chaque ligne rendue (index logique 1-based), spacers aria-hidden.

<div class="virtual-list" data-vlist-count="1000">
  <div class="virtual-list-viewport"></div>
</div>

Graphe

Graph

Moteur graphique node-link maison — pipeline measure → layout → paint, layouts fixed, tree, radial et mindmap purs (DOM-free) plus layered (Sugiyama via dagre vendoré, seul layout asynchrone), auto-détection layout: 'auto' qui route vers tree ou layered selon la topologie, viewport pan/zoom/pinch (molette ancrée curseur, glisser, pincer tactile, bornée --graph-zoom-min/-max), fit-to-content et sélection nœud/arête (clic, clavier, ResizeObserver optionnel), nav clavier nœud-à-nœud (roving tabindex + traversal via un arbre couvrant déterministe, mapping WAI-ARIA APG tree : flèches/Home/End/Enter), alternative accessible sous forme de tableau reliée au SVG par aria-describedby.

Démo — organigramme (layout fixed)

size explicite sur chaque nœud — la passe measure est skippée, géométrie déterministe (VR stable).

Démo — arbre (layout tree, Reingold-Tilford)

Racine = nœud sans arête entrante, enfants via outEdges, déterministe (mêmes entrées → même géométrie).

Démo — viewport + sélection (fit, clic nœud/arête, clavier)

Molette : zoom centré curseur · glisser : pan · pincer : zoom tactile · clic nœud/arête : sélection (halo) + détail modal · f fit · +/- zoom · Échap désélection. initialViewport et initialSelection figent un état déterministe (VR stable) — le halo est visible sans interaction, le modal ne s'ouvre jamais au chargement. Arêtes en vector-effect:non-scaling-stroke (épaisseur constante au zoom).

Démo — radial (layout radial, éventail 360°)

Racine au centre, profondeur → anneau, secteurs ∝ feuilles. size explicite → mesure skippée, VR déterministe.

Démo — dépendances (layout layered, Sugiyama via dagre vendoré)

DAG multi-racines (spec+docs) — couches Sugiyama via @dagrejs/dagre vendoré (dynamic import, hors bundle de base). layered est le seul layout asynchrone du moteur (run() renvoie une Promise) — paint() tolère le thenable, token anti-course.

Démo — mindmap NHOOD (layout mindmap, bilatéral)

1er use case client (NHOOD) : nœud central → branches réparties gauche/droite (glouton, équilibrage par charge de sous-arbre) → feuilles riches (graphCard() — badge statut + chip personne dans un foreignObject). layout:'mindmap' reste toujours explicite (jamais auto-détecté). size explicite sur chaque nœud → VR déterministe.

Usage

Pipeline : measure (tailles effectives dans une Map interne au renderer — node.size si fourni, sinon mesuré ; le modèle n'est jamais muté) → layout ({x,y} purs, DOM-free) → paint (couleurs 100 % var(--graph-*) — repaint gratuit au toggle theme/mode).

Cycle de vie : model.addEventListener('graph:model:change', ...) → repaint débouncé via requestAnimationFrame → destroy() enregistré via window.__registerInstance (teardown SPA, #657).

Viewport : opts.viewport (bool, défaut true), opts.zoomMin/opts.zoomMax (override --graph-zoom-min/-max), opts.initialViewport ({tx,ty,k} état déterministe). API retournée : getViewport()/setViewport(v)/screenToWorld(cx,cy), événement graph:viewport:change (detail:{tx,ty,k}) sur le conteneur .graph. Un ancêtre en transform:scale() casse getScreenCTM() — interdit côté consumer.

Fit : fit() ramène le viewport à l'identité ({tx:0,ty:0,k:1}) — le viewBox posé par paint() cadre déjà le contenu, l'identité EST le fit-to-content. zoomToNode(id, k=1.5) centre + zoome sur un nœud. opts.refitOnResize (bool, défaut false) re-cadre au redimensionnement du conteneur via ResizeObserver (débouncé rAF, teardown dans destroy()) — UNIQUEMENT si l'utilisateur n'a pas déjà navigué (ne casse jamais une vue existante). Le responsive de base est déjà assuré par le viewBox, sans JS.

Sélection : opts.selectable (bool, défaut true) — clic nœud/arête → classes .graph-node--selected/.graph-edge--selected + focus + événement graph:selection:change (detail:{id,kind}) sur .graph — même canal que graph:viewport:change. Détail au clic via opts.onSelect (callback) sinon modal DS par défaut (opts.selectionDetail:false pour désactiver). API retournée : select(id|null)/getSelection(). opts.initialSelection fige un état déterministe (VR) — pose le halo sans ouvrir le détail. La sélection est un concern du renderer, pas du GraphModel (données pures) — pré-requis de l'édition (I5). Clavier : Échap désélectionne, f fit, +/- zoom centré, flèches pan (conteneur tabindex="0"). Nav nœud-à-nœud (roving tabindex, un seul nœud tabindex="0" à la fois) : ↑ parent, ↓ 1er enfant, ←/→ frère précédent/suivant, Home/End premier/dernier nœud, Entrée/Espace sélectionne — traversal via un arbre couvrant déterministe (buildSpanningTree()), flèches sur un nœud ne déclenchent jamais le pan conteneur. opts.keyboardNav (bool, défaut true) désactive cette nav.

Alternative a11y : .graph-table reliée au SVG par aria-describedby, reconstruite à chaque repaint (contrat primaire, WCAG 1.1.1/1.3.1).

</div>
