Usage

Utiliser btn-primary pour l'action principale d'une page ou d'un formulaire — un seul par section. Utiliser btn-secondary pour les actions secondaires et btn-ghost pour les actions tertiaires ou de navigation. Card cliquable (a11y)

Wrapper card-link pour rendre une card entierement cliquable. Focus-visible et hover geres sur la card enfant. Semantiquement un lien <a>.

Projet Aksyva

Cliquer pour ouvrir le detail du projet.

Projet Acssi

Usage

Utiliser card pour les contenus richement structures (titre + description + actions). Utiliser card-compact pour les grilles de features ou les KPI courts. Utiliser card-horizontal pour les listes avec icone + texte sur une seule ligne. Utiliser card-link (wrapper <a>) pour rendre une card entierement cliquable avec focus-visible a11y. Ne pas imbriquer des cards — preferer une demo-box ou une subsection pour les conteneurs secondaires.

Badge compact (sidebar/nav)

Variante badge-nav pour les compteurs/percentages dans une sidebar ou item de nav. margin-left:auto push-right, min-width:1.5rem stable.

Inbox 12
Notifications 3
Done 98%
Usage

Utiliser les badges pour indiquer un statut (déployé, en attente, erreur) ou une catégorie sur un élément de liste ou une card. badge-success pour les états positifs, badge-warning pour les alertes non-critiques, badge-danger pour les erreurs et badge-info pour les informations neutres. Utiliser badge-nav pour les compteurs compacts dans une sidebar ou nav (push-right via margin-left:auto). Ne pas utiliser des badges pour des actions — préférer des boutons. Limiter à 3-4 badges visibles simultanément pour éviter la surcharge visuelle.

Reset natif

Resets globaux pour les elements user-agent illisibles en dark mode. Conforme WCAG 1.4.3 (contraste liens ≥ 4.5:1) et WCAG 2.4.7 (focus visible ≥ 3:1, ≥ 2px). Consolide closes #166 + closes #167.

Liens natifs

Voici un lien natif sans classe dans un paragraphe.

Lien externe : msyx.fr. Lien avec texte plus long : documentation du design system.

Avant le reset : couleur user-agent #0000ee — ratio 1.35:1 sur ACSSI dark (invisible). Apres : var(--accent) theme-aware ≥ 4.5:1 dans tous les themes.

Survol : text-decoration-thickness: 2px (feedback non-couleur WCAG 1.4.1). Focus clavier (Tab) : outline 2px accent.

Focus visible global

Naviguer au clavier (Tab) sur les elements ci-dessous — outline 2px var(--accent), offset 2px. Les composants DS avec focus custom (.btn-*, .input, etc.) gardent leur apparence propre.

A11y

Disabled global

Règle CSS globale pour styliser l'état disabled des éléments natifs non couverts par .btn-* ou .input. Promue depuis aksy DS-EXCEPTION (#301). Couvre : select, textarea, input[type=checkbox/radio], boutons et liens sans classe DS, lignes tr, etc.

Éléments natifs [disabled]

Opacité 0.5, cursor not-allowed — discernables de l'état idle dans tous les thèmes.

Lien actif Lien aria-disabled

a[aria-disabled="true"] → opacité 0.5, pointer-events none.
Les .sidebar-link[aria-disabled] (spécificité 0,2,0) ne sont pas affectés.

Sélecteur exclusion : les classes DS .btn-* et .input gardent leur propre gestion disabled.
