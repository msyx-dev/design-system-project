Usage

Utiliser btn-primary pour l'action principale d'une page ou d'un formulaire — un seul par section. Utiliser btn-secondary pour les actions secondaires et btn-ghost pour les actions tertiaires ou de navigation. Ne pas utiliser btn-danger pour des actions sans confirmation — preferer un modal de confirmation. Pour les actions icones seules, utiliser btn-icon avec un aria-label obligatoire.

Groupe de boutons vs Segmented control : utiliser .btn-group (CSS-only, role="group") pour accoler des actions distinctes (ex. Principe WCAG — contraste preserve (#569)

L'opacity 0.85 s'applique au conteneur (chrome : fond, bordure, icone) mais pas au texte lui-meme. La couleur var(--text) reste a 100% d'opacite effective car CSS composite l'opacity du parent uniquement sur la surface rendue, preservant le ratio de contraste ≥ 4.5:1 requis par WCAG 1.4.3. Valider : DevTools > Accessibility > Contrast.

--opacity-muted0.85 — token global (tokens.css)
.card-muted .card-iconopacity:0.6 — icone plus attenuee que le chrome
.card-muted:hoveropacity:1 — remontee complete au survol
Usage

Utiliser card pour les contenus richement structures (titre + description + actions). Utiliser card-compact pour les grilles de features ou les KPI courts. Utiliser card-horizontal pour les listes avec icone + texte sur une seule ligne. Utiliser card-link (wrapper <a>) pour rendre une card entierement cliquable avec focus-visible a11y. Utiliser card-muted pour les cartes sous un seuil de pertinence — contraste WCAG AA garanti via opacity globale 0.85 sans toucher la couleur texte. Ne pas imbriquer des cards — preferer une demo-box ou une subsection pour les conteneurs secondaires.

Disabled global

Règle CSS globale pour styliser l'état disabled des éléments natifs non couverts par .btn-* ou .input. Promue depuis aksy DS-EXCEPTION (#301). Couvre : select, textarea, input[type=checkbox/radio], boutons et liens sans classe DS, lignes tr, etc.

Éléments natifs [disabled]

Opacité 0.5, cursor not-allowed — discernables de l'état idle dans tous les thèmes.

Lien actif Lien aria-disabled

Sélecteur exclusion : les classes DS .btn-* et .input gardent leur propre gestion disabled.

Split Button

Action principale + caret menu contextuel attache. Le bouton primaire declenche l'action par defaut ; le caret ouvre un panneau d'alternatives bati sur le primitif .menu. Navigation clavier complete (fleches, Home/End, Echap). initSplitButton() dedie.

Variante Primary
Variante Secondary
Etat desactive
Usage

Utiliser .split-button pour proposer une action principale evidente (bouton de gauche) avec des alternatives contextuelles (caret de droite). Le caret porte aria-haspopup="menu" + aria-expanded synchronise. Le panneau est bati sur le primitif .menu (#520). Ne pas confondre avec .btn-group (groupement d'actions distinctes, CSS-only, pas de menu).
