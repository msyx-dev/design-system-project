Getting Started

Integrer le design system msyx.design en 3 etapes — tokens, composants, theming.

Pourquoi ce design system ?

Le DS msyx.design est la source de verite unique pour tous les projets msyx.fr. Il garantit la coherence visuelle, le respect du theming (3 palettes × 2 modes) et l'accessibilite WCAG AA sur tous les projets consommateurs.

Getting Started

Installation

Trois niveaux d'integration selon vos besoins.

Etape 1 — Synchroniser les fichiers CSS

Depuis le repertoire shared/ du design system, lancer le script de sync :

./sync.sh /chemin/vers/votre-projet/src/styles/

Cela copie 4 fichiers prefixes ds- dans votre dossier styles.

Niveau A — Tokens seuls (minimal)

Acces aux variables CSS uniquement. Ideal si vous avez votre propre systeme de composants.

@import url('./styles/ds-tokens.css');

Donne acces a var(--accent), var(--surface), var(--text-muted), spacing, radius, shadows...

Niveau B — CSS complet (recommande)

Tokens + classes utilitaires + composants. Sans le layout shell (header/sidebar).

@import url('./styles/ds-tokens.css'); @import url('./styles/ds-utilities.css'); @import url('./styles/ds-components.css');

Donne acces a toutes les classes : .btn-primary, .card, .badge-success, .text-muted...

Niveau C — CSS + JS (shell complet)

Integration complete avec layout, composants interactifs (modals, toasts, sliders...) et header avec gestion utilisateur.

<!-- Dans <head> --> @import url('./styles/ds-tokens.css'); @import url('./styles/ds-utilities.css'); @import url('./styles/ds-layout.css'); @import url('./styles/ds-components.css'); <!-- En fin de <body> --> <script src="/shared/nav.js"></script> <script src="/shared/components.js"></script>

Premiers pas

Structure HTML de base et configuration du theming.

Structure HTML de base

Template minimal pour une page utilisant le shell DS (header + sidebar + contenu) :

<!DOCTYPE html> <html lang="fr"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Ma page — Mon Projet</title> <link rel="stylesheet" href="/shared/styles.css"> <!-- Anti-FOUC : appliquer le theme AVANT le rendu --> <script>(function(){ var t = localStorage.getItem('msyx-theme'); if (t && t !== 'msyx') document.documentElement.setAttribute('data-theme', t); var m = localStorage.getItem('msyx-mode'); if (m && m !== 'dark') document.documentElement.setAttribute('data-mode', m); }());</script> </head> <body> <header class="site-header" id="site-header"></header> <aside class="sidebar" id="sidebar"></aside> <div class="main"> <section id="ma-section"> <div class="section-header"> <span class="overline">Categorie</span> <h2>Titre de section</h2> <p>Description de la section.</p> </div> <!-- Contenu ici --> </section> </div> <script src="/shared/nav.js"></script> <script src="/shared/components.js"></script> </body> </html>
Configurer le theme

Deux attributs HTML controlent l'apparence. Les placer sur <html> :

<!-- Theme par defaut --> <html data-theme="msyx" data-mode="dark"> <!-- Corporate or (ACSSI) --> <html data-theme="acssi" data-mode="dark"> <!-- Corporate vert (Nhood) --> <html data-theme="nhood" data-mode="dark"> <html data-theme="nhood" data-mode="light">
msyx : dark + light acssi : dark only nhood : dark + light

Header avec utilisateur

Configurer la zone utilisateur, les notifications et le menu du header.

Configuration window.MSYX_HEADER

Definir cet objet avant le chargement de nav.js pour activer la zone utilisateur :

<script> window.MSYX_HEADER = { auth: true, user: { name: 'Mike', initials: 'M', avatar: '/img/avatar.png' // optionnel }, notifications: { enabled: true, count: 3 }, menu: [ { label: 'Profil', icon: '&#128100;', href: '/profil' }, { label: 'Preferences', icon: '&#9881;', href: '/settings' }, { divider: true }, { label: 'Deconnexion', icon: '&#128682;', action: 'logout', class: 'danger' } ] }; </script> <script src="/shared/nav.js"></script>

Sans window.MSYX_HEADER ou avec auth: false : header logo + theme switcher uniquement.

Utiliser les tokens

Variables CSS disponibles et classes utilitaires.

Tokens principaux
/* Couleurs */ var(--accent) /* Couleur accent du theme actif */ var(--surface) /* Surface de carte/panel */ var(--text) /* Texte principal */ var(--text-muted) /* Texte secondaire */ var(--text-dim) /* Texte desactive */ var(--border) /* Bordure standard */ /* Semantiques */ var(--success) var(--warning) var(--danger) var(--info) /* RGB pour rgba() */ rgba(var(--accent-rgb), 0.1) rgba(var(--success-rgb), 0.2) /* Spacing */ var(--space-xs) /* 0.25rem */ var(--space-sm) /* 0.5rem */ var(--space-md) /* 1rem */ var(--space-lg) /* 1.5rem */ var(--space-xl) /* 2rem */ var(--space-2xl) /* 3rem */
Classes utilitaires
/* Texte */ .text-muted .text-dim .text-accent .text-success .text-warning .text-danger .text-info /* Backgrounds */ .bg-surface .bg-accent .bg-success .bg-warning .bg-danger .bg-info /* Bordures */ .border-default .border-accent .border-subtle /* Accessibilite */ .sr-only /* Visible seulement par les lecteurs d'ecran */ .focus-visible /* Focus ring visible au clavier */

Bonnes pratiques

Ce qu'il faut faire et ce qu'il faut eviter.

A eviter
color: #94a3b8; background: #1e293b; border: 1px solid rgba(59,130,246,0.2); style={{ color: "var(--text-muted)" }} style={{ background: "var(--accent)" }}
Recommande
color: var(--text-muted); background: var(--surface); border: 1px solid rgba(var(--accent-rgb), 0.2); className="text-muted" className="bg-accent"
Regle d'or

Jamais de composant reimplemente hors DS. Si un composant manque, ouvrir une issue avec le label composant sur le repo design-system-project. Le composant est cree dans le DS, puis consomme via sync.sh.
