Theming

Infrastructure 2 axes : data-theme pour la palette (MSYX, ACSSI…) et data-mode pour la luminosite (dark / light). Les deux attributs se posent sur la balise <html>.

Architecture 2 axes
/* Axe 1 — palette */
<html data-theme="acssi">

/* Axe 2 — luminosite */
<html data-mode="light">

/* Combinaison */
<html data-theme="msyx" data-mode="light">
Cascade 4 couches
1:root — valeurs dark MSYX par defaut
2[data-theme] — surcharge palette
3[data-mode="light"] — surcharge luminosite
4[data-theme][data-mode] — combinaison fine
Variables themables
Surfaces — --primary, --surface, --surface-solid
Texte — --text, --text-muted, --text-dim
Accents — --accent, --deco-violet, --deco-cyan, --deco-pink
Semantique — --success, --warning, --danger, --info
Charts — --chart-1 a --chart-5
Syntax — --code-keyword, --code-string, etc.
Sidebar — --sidebar-bg, --sidebar-link-hover-bg, --sidebar-link-active-bg
Modes disponibles par theme
MSYXdark + light
Palette bleu profond — les deux modes disponibles via le toggle lune/soleil.
ACSSIdark only
Palette bleu marine/or — le bouton light est grise (dark-only). Extensible en v2.
Futur themedark + light
Ajouter dans THEME_CONFIG : { modes: ['dark','light'], defaultMode: 'dark' }
Persistance — 2 cles localStorage
/* Cle 1 : palette */
localStorage.setItem('msyx-theme', 'acssi');

/* Cle 2 : mode */
localStorage.setItem('msyx-mode', 'light');

/* Valeurs par defaut (absentes) */
/* theme = 'msyx' | mode = 'dark' */
Anti-FOUC — script inline <head>

Charge les deux attributs avant le premier rendu pour eviter le flash de contenu non style.

<script>(function(){
  var t=localStorage.getItem('msyx-theme');
  if(t&&t!=='msyx')
    document.documentElement.setAttribute('data-theme',t);
  var m=localStorage.getItem('msyx-mode');
  if(m&&m!=='dark')
    document.documentElement.setAttribute('data-mode',m);
}());</script>

Design Rule

Performance & Glassmorphism

Regle « glass for chrome, solid for content » — quand utiliser backdrop-filter et quand le proscrire.

Chrome UI (glass)Header, sidebar, toasts — surfaces navigationnelles persistantes, petites zones
Content (solid)Modals dialogues longs, tables data-heavy, drawers pleins ecrans — lisibilite prioritaire
@supports fallbackbackdrop-filter absent (Firefox flag off) → background: var(--surface-solid) + opacity 0.95

Le glassmorphism renforce l'identite visuelle MSYX sur les navigateurs recents. Pour les contextes de contenu dense (tableaux, formulaires longs, modals de confirmation avec texte), privilegier var(--surface-solid) pour maximiser le contraste WCAG AA.

Texture grain

Layer noise SVG appliqué globalement via body::after. Ajoute une signature visuelle subtile sur tout le DS — perçu comme « profondeur » même en grayscale.

Tokens (v2.42.0)
--texture-grainURL SVG inline — filtre feTurbulence 0.9 Hz, 4 octaves, stitch
--texture-grain-opacity0.015 — subtil, perceptible sur fond uni en lumière directe
Usage
body::after {
  content: '';
  position: fixed; inset: 0;
  pointer-events: none; z-index: 9999;
  opacity: var(--texture-grain-opacity);
  background-image: var(--texture-grain);
}
Principe

Le grain est positionné en fixed avec z-index: 9999 et pointer-events: none — invisible aux interactions, présent visuellement sur toutes les couches de l'UI. Identité visuelle reconnaissable même sur impression monochrome ou capture d'écran.

SVG theme-aware

Pattern pour que les SVG inlines (data:uri ou inline <svg>) suivent le theme et le mode actif du design system.

Probleme

Les data:uri utilises dans background-image ne resolvent pas currentColor ni les variables CSS — la couleur est figee a la compilation. Resultat : un chevron stroke="#94a3b8" reste gris fixe sur tous les themes, causant un contraste insuffisant sur ACSSI dark et une visibilite reduite en mode light (P-02 audit Phase 1).

Solution — variable CSS contenant le data:uri complet
/* tokens.css — 6 declinaisons theme/mode */
:root {
  --chevron-select: url("data:image/svg+xml,%3Csvg...stroke='%237c8db5'...");
}
[data-mode="light"] { --chevron-select: url("...stroke='%2364748b'..."); }
[data-theme="acssi"] { --chevron-select: url("...stroke='%239eb3c8'..."); }
[data-theme="acssi"][data-mode="light"] { --chevron-select: url("...stroke='%234a6a84'..."); }
[data-theme="nhood"] { --chevron-select: url("...stroke='%236a9e7a'..."); }
[data-theme="nhood"][data-mode="light"] { --chevron-select: url("...stroke='%235a8a68'..."); }
Usage dans le composant
select.input {
  background-image: var(--chevron-select); /* theme-aware via token */
}
Antipattern
/* NE PAS FAIRE — couleur hardcodee, ne s'adapte pas au theme */
select.input {
  background-image: url("data:image/svg+xml,...stroke='%2394a3b8'...");
}
Principe

La couleur est encodee directement dans l'URL SVG (# → %23). La variable CSS --chevron-select est definie en miroir de --text-dim dans chaque bloc theme/mode de tokens.css. Ce pattern est extensible : tout SVG inline peut etre tokenise de cette facon. (v2.44.0)
