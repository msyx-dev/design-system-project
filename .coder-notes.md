## Divergences avec la spec

- **`@axe-core/playwright` API** : la spec demandait `injectAxe`/`checkA11y` (API du package `axe-playwright`), mais `@axe-core/playwright` v4.x (package officiel Deque) expose uniquement `AxeBuilder`. L'implémentation utilise l'API correcte `new AxeBuilder({ page }).withTags([...]).analyze()`.
- **Config dédiée** : la spec demandait `--project=chromium` dans le workflow, mais `playwright.config.ts` définit 12 projets nommés `<theme>-<mode>-<viewport>`. Création de `playwright.a11y.config.ts` pour isoler proprement les 54 runs (sans les multiplier × 12 projets VR).
- **PORT serveur** : la spec mentionnait port 3000, mais port 4173 déjà pris par le pipeline VR. Choix du port 3001 dans `playwright.a11y.config.ts` avec `webServer` intégré.
- **RELEASES.md** : le hook `hook-block-coder-mutations.py` bloque l'écriture directe de RELEASES.md pendant `/dev`. Fragment préparé dans la description PR (convention pipeline).

## Résultat du dry-run

**54 runs / 54 passés — 0 violations WCAG 2.0/2.1 A/AA**

CSS bien chargé vérifié : `--primary: #0a0f1e` retourné. Le DS utilise HTML sémantique + patterns ARIA documentés.
