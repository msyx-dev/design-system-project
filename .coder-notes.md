## Points d'attention pour le tester

1. `build.sh` necessite `terser` et `csso-cli` (npm install -g terser csso-cli). Tester
   d'abord avec `--check-only` pour verifier les outils avant de lancer la minification.

2. `check-sync.sh` : tester les 2 formes — avec repertoire (`check-sync.sh /path/styles/`)
   et avec fichier direct (`check-sync.sh /path/styles/ds-tokens.css`) — les deux doivent fonctionner.

3. `sync-all.sh` requiert jq. Si absent : le script affiche un message d'installation explicite.

4. Le repertoire `shared/dist/` n'est pas commite — il est cree par build.sh. Ne pas
   commiter un dist/ vide manuellement.
