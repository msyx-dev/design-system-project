## Divergences avec la spec

- **Version** : la spec indiquait 2.14.4 (patch), mais `tokens.css` était déjà à 2.15.0 (commit précédent). J'ai aligné à 2.15.1 pour toutes les ressources CSS. C'est cohérent avec le semver existant.
- **Sidebar filter — anti-double-bind** : la fonction `bindSidebarFilter()` est appelée depuis `buildSidebar()` qui recrée le DOM à chaque appel SPA. Le `dataset.bound` sur l'input suffit à éviter un double-bind si `buildSidebar()` est rappelé sans recréer le DOM, mais en pratique `buildSidebar()` recrée toujours le HTML complet → pas de risque de double-bind réel.

## Points d'attention pour le tester

1. **Command palette — Cmd+K vs Ctrl+K** : le raccourci fonctionne avec les deux. Sur Mac, tester ⌘K. Sur PC, tester Ctrl+K.
2. **Command palette — singleton** : le listener global `keydown` est protégé par `document.body.dataset.cmdPaletteBound`. Si `initComponents()` est appelé plusieurs fois (SPA), la palette ne se duplique pas et le listener n'est pas re-bindé.
3. **Filtre sidebar — sections vides** : l'algorithme parcourt les frères DOM pour détecter si une section a des liens visibles. Si la structure HTML de la sidebar évolue (ex. nouveaux éléments intercalés), revoir `bindSidebarFilter()`.
4. **Ctrl+F sur site.html** : le listener ne fait pas `preventDefault()` — le comportement natif du navigateur s'ouvre normalement APRÈS le chargement des sections. Les sections déjà chargées ne sont pas rechargées (protection `loadedSections`).
5. **Actions command palette — toggle mode** : lit `data-mode` sur `<html>`. Ne passe pas par `initModeSwitcher()` — si la logique de mode change dans le futur, synchroniser.
6. **Index command palette** : construit à l'init, pas de mise à jour dynamique si NAV_SECTIONS change en cours de session.
